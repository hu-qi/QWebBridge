Everything an AI agent needs to browse the web, interact with pages, and extract information.

navigate

Navigate to any URL. Opens new tabs with session-based grouping and auto-load waiting.

📸

⚡

Run arbitrary JavaScript in the page. Supports async/await.

🖱️

Real mouse event dispatch at element coordinates.

⌨️

key_type

Type text character by character via CDP.

🔑

Keyboard shortcuts with modifier keys — Control, Alt, Shift, Meta.

📎

Upload files to inputs. Supports multiple files.

🌐

network

Monitor network requests — start, stop, list, inspect.

🔍

find_tab

Find an open tab by URL. Target active tab or first match.

📑

list_tabs

List all open tabs with URL, title, and session group.

❌

Close the current tab or a specific tab by ID.

🧹

close_session

📄

Integration

Works with any AI agent

Connect your preferred coding agent through the protocol that fits your workflow.

WebSocket

WebSocket API

Kimi WebBridge compatible protocol. Connect any agent that speaks WebSocket.

ws://localhost:10086/selector/command
MCP

Model Context Protocol

Standard MCP server for Claude Desktop, Cursor, Windsurf, and any MCP client.

qweb-bridge mcp
REST

HTTP API

Simple HTTP POST interface for custom scripts and integrations.

POST /api/tool/<name>
CLI

Command Line

Direct terminal access for quick browser automation from your shell.

qweb-bridge <command>
Install

Get started in 1 minute

One-command install. Prerequisites: Chrome browser and Node.js ≥ 18.

Quick install — click to copy
curl -fsSL https://raw.githubusercontent.com/hu-qi/QwebBridge/main/install.sh | bash

Also available via npm, pnpm, and manual build — see tabs below.

# One-command install (macOS / Linux)
curl -fsSL https://raw.githubusercontent.com/hu-qi/QwebBridge/main/install.sh | bash
# Install with npm (coming soon)
npm install -g qweb-bridge
# Install dependencies
pnpm install

# Build all packages
pnpm build
$ git clone https://github.com/hu-qi/QwebBridge.git
$ cd QwebBridge
$ pnpm install
$ pnpm build
$ node packages/daemon/dist/cli.js run

Setup steps

  1. 1
    Start the daemon

    Run qweb-bridge start or node packages/daemon/dist/cli.js run. Listens on localhost:10086.

  2. 2
    Install the Chrome extension

    Open chrome://extensions, enable Developer mode, click "Load unpacked", select packages/extension/dist.

  3. 3
    Connect your AI agent

    Point to ws://localhost:10086/selector/command (WebSocket) or run qweb-bridge mcp (MCP).

  4. 4
    Verify

    Run qweb-bridge status — look for extensions_connected: true.

✓ Chrome browser ✓ Node.js ≥ 18 ✓ pnpm ✓ macOS / Linux / Windows
FAQ

Frequently asked questions

How does QwebBridge work?▾
QwebBridge pairs a local daemon with a Chrome extension. The AI agent sends commands via WebSocket, MCP, or HTTP to the daemon on localhost:10086, which forwards them to the Chrome extension. The extension uses Chrome DevTools Protocol (CDP) to control the browser. Everything runs locally on your machine.
What's the difference from Kimi WebBridge?▾
QwebBridge is open-source and protocol-compatible with Kimi WebBridge. Any agent that works with Kimi WebBridge can use QwebBridge seamlessly. QwebBridge additionally supports MCP, HTTP REST API, and a full CLI.
Which AI agents are supported?▾
Works with Claude Code, Kimi Code, Cursor, GitHub Copilot (Codex), Hermes, OpenClaw, and any custom agent that supports WebSocket or MCP.
Is my data safe?▾
Yes. The daemon and extension communicate over localhost only. No browser data, login sessions, cookies, or page content ever leaves your machine.
How do I install QwebBridge?▾
Run curl -fsSL https://raw.githubusercontent.com/hu-qi/QwebBridge/main/install.sh | bash for one-command install, or clone the repo and run pnpm install && pnpm build.
What protocols does QwebBridge support?▾
Four protocols: WebSocket (ws://localhost:10086/selector/command), MCP (qweb-bridge mcp), HTTP REST (POST /api/tool/<name>), CLI (qweb-bridge <command>).
Can I use QwebBridge without an AI agent?▾
Yes. Use the HTTP REST API directly (curl -X POST http://localhost:10086/api/tool/screenshot) or the CLI (qweb-bridge snapshot) from scripts and cron jobs.
